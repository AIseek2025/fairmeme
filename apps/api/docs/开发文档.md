```
├─cmd  //程序入口目录

    └─build
```

所有业务逻辑只在`internal`文件夹进行，下面是定义一个`solana`相关接口的步骤。

1. 在`routes`文件夹定义一个路由，按业务划分文件进行，单独创建`solana.go`，内容可以复制其他文件内容进行修改。
2. 在`handler`文件夹创建`solana.go`,使用`interface`方式进行开发

```
type SolanaHandler interface {
	Create(c *gin.Context)
	UpdateByID(c *gin.Context)
	GetByAddress(c *gin.Context)
	List(c *gin.Context)
}

//下面是一个使用多个dao的示例
type solanaHandler struct {
	iDao       dao.TokenDao
	holdersDao dao.HoldersDao
}

// NewSolanaHandler creating the handler interface
func NewSolanaHandler() SolanaHandler {
	return &solanaHandler{
		iDao: dao.NewTokenDao(
			model.GetDB(),
			cache.NewTokenCache(model.GetCacheType()),
		),
		holdersDao: dao.NewHoldersDao(
			model.GetDB(),
			cache.NewHoldersCache(model.GetCacheType()),
		),
	}
}
```

3. 定义业务逻辑时，数据库方法在`dao`文件夹，`dao`文件夹内文件和`model`文件夹内文件一一对应，定义方法时在对应表名的文件内定义。
4. 不查询数据库时，在`service`文件夹定义，创建`solana.go`进行开发
5. 需要使用`redis`需要在`cache`定义方法，比如`clickhouse`需要使用`redis`,新建`clickhouse.go`定义方法进行调用。

## 代码规范

1. 遵循单一职责原则，根据业务逻辑写在相应的文件。
2. 变量名全部为小驼峰，严谨变量名与软件包名同名，不允许对软件包名重命名。类名使用大驼峰。
3. 接口遵循`RESTful`规范，路由命名根据 API版本、模块、功能进行划分，比如获取`Solana`价格，可以定义为`api/v1/solana/getPrice`
   。获取`Toekn`价格，定义为`api/v1/token/getPrice`。需要在对应路由文件内定义。
4. `handle`尽可能只作为中转使用，不允许超过3层嵌套if，嵌套if需要进行拆分。
5. 错误需要记录到日志，关键业务需要记录到日志。
```
//错误日志
logger.Error()
//业务日志
logger.Info()
```
